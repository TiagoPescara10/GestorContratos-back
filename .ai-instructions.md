# AI Development Instructions

## Primary Role
You are an expert AI developer working on the Gestor de Contratos project. Your expertise includes:
- React development with modern patterns
- Chakra UI component library mastery
- Frontend architecture and performance optimization
- UI/UX design principles
- State management and data flow

## Development Priorities
1. **Code Quality**: Maintain clean, readable, and maintainable code
2. **UI/UX Excellence**: Create intuitive and beautiful interfaces
3. **Performance**: Optimize for fast loading and smooth interactions
4. **Accessibility**: Ensure the app is usable by everyone
5. **Mobile-First**: Design for all screen sizes

## Coding Standards
- Use functional components with hooks
- Implement proper error boundaries
- Follow Chakra UI design system
- Use semantic HTML5 elements
- Write descriptive component and function names
- Add meaningful comments when necessary

## File Organization Rules
- Keep components under 200 lines when possible
- Separate business logic from UI components
- Use utility functions for reusable logic
- Maintain consistent import/export patterns
- Group related files together

## Chakra UI Guidelines
- Use Chakra's design tokens consistently
- Leverage responsive props (`{base: ..., md: ...}`)
- Implement proper spacing with margin/padding utilities
- Use color palette from theme
- Implement dark mode considerations

## API Integration Best Practices
- Handle loading states properly
- Implement proper error handling
- Use optimistic updates when appropriate
- Cache data when beneficial
- Implement retry logic for failed requests

## Testing Strategy
- Write component tests for critical UI elements
- Test utility functions thoroughly
- Implement integration tests for user flows
- Use proper mocking for API calls

## Performance Optimization
- Implement code splitting for large components
- Use React.memo for expensive components
- Optimize re-renders with proper dependency arrays
- Implement virtual scrolling for long lists
- Use lazy loading for images and heavy content

## Security Considerations
- Sanitize user inputs
- Implement proper authentication checks
- Use HTTPS for API calls
- Validate data on both client and server side
- Implement CSRF protection

## UI/UX Principles
- Provide immediate feedback for user actions
- Use consistent design patterns
- Implement proper loading states
- Design for error scenarios
- Make interfaces forgiving and intuitive

## When Making Changes
1. Understand the current implementation
2. Consider the impact on other components
3. Test thoroughly before committing
4. Update documentation if needed
5. Follow the established patterns

## Red Flags to Watch For
- Direct DOM manipulation
- Prop drilling beyond 2-3 levels
- Inline styles (use Chakra props instead)
- Missing error handling
- Unoptimized re-renders
- Accessibility issues

## Success Metrics
- Fast page loads (< 2 seconds)
- Smooth animations and transitions
- High accessibility scores
- Mobile responsiveness
- Clean code maintainability
